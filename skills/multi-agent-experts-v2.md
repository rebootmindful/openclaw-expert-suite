## 二、问题复杂度评估算法

### 评估维度（1-10分制）

```
复杂度总分 = 领域跨度×2 + 抽象程度×1.5 + 决策难度×2 + 时间跨度×1.5 + 不确定性×2

等级划分：
- 🟢 简单（<20分）：单专家即可
- 🟡 中等（20-35分）：2位专家
- 🔴 复杂（35-50分）：3-4位专家
- ⚫ 极复杂（>50分）：4位+迭代优化
```

### 维度详解

| 维度 | 评估标准 | 低分示例 | 高分示例 |
|------|---------|---------|---------|
| **领域跨度** | 涉及多少个专业领域 | "如何学习Swift" | "AI+生物技术的伦理边界" |
| **抽象程度** | 是否需要哲学/本质思考 | "这个API怎么用" | "什么是智能的本质" |
| **决策难度** | 是否涉及重大选择 | "推荐一本书" | "是否应该辞职创业" |
| **时间跨度** | 影响周期长短 | "今天吃什么" | "未来10年职业规划" |
| **不确定性** | 信息完整度 | "Python语法" | "预测下一个技术趋势" |

## 三、动态专家匹配算法

### 第一步：问题解析

提取以下特征：
- **关键词集合**（技术/商业/哲学/个人）
- **问题类型**（what/how/why/should）
- **时间维度**（过去/现在/未来）
- **主观程度**（客观事实 vs 主观决策）

### 第二步：复杂度评分

根据五个维度计算总分，确定复杂度等级。

### 第三步：专家选择策略

#### 🟢 简单级（<20分）

**策略**：单专家，快速响应

```
领域关键词 → 匹配专家
- 技术/工具 → Researcher
- 本质/哲学 → Thinker
- 迷茫/困惑 → Coach
- 决策/选择 → Decision
- 方法/框架 → Methodology
- 成长/评估 → Human3
- 财富/策略 → Naval
- 命运/文化/破局 → Tiandao
- 规划/数据/系统 → Strategist
```

#### 🟡 中等级（20-35分）

**策略**：双专家，互补视角

**黄金组合库**（基于历史反馈）：

| 场景特征 | 推荐组合 | 成功率 |
|---------|---------|--------|
| 技术+本质 | Researcher + Thinker | 92% |
| 决策+知识 | Decision + Researcher | 88% |
| 个人成长 | Human3 + Coach | 90% |
| 方法论 | Methodology + Thinker | 85% |
| 战略+财富 | Decision + Naval | 87% |
| 命运破局 | Tiandao + Thinker | 90% |
| 强者思维 | Tiandao + Coach | 88% |
| 个人战略规划 | Strategist + Human3 | 89% |
| 效率优化 | Strategist + Methodology | 86% |

#### 🔴 复杂级（35-50分）

**策略**：3-4位专家，全面覆盖

**自动组合公式**：
```
必选：Researcher（领域知识）
必选：Thinker（本质挖掘）
+ 根据类型选择1-2位：
  - 决策类 → Decision
  - 个人类 → Human3 + Coach
  - 策略类 → Naval
  - 方法类 → Methodology
  - 命运困局类 → Tiandao + Decision
  - 长期规划类 → Strategist + Naval
```

#### ⚫ 极复杂级（>50分）

**策略**：分轮次派遣，迭代深化

**第一轮**（基础研究）：
- Researcher：领域全景扫描
- Thinker：核心矛盾识别
- Tiandao：文化属性诊断（如涉及命运/格局问题）

**第二轮**（深度分析）：
- 根据第一轮结果，针对性派遣
- 如需要决策模拟 → Decision
- 如需要方法论 → Methodology
- 如需要长期规划 → Strategist

**第三轮**（综合与验证）：
- Coach：检视思维盲区
- 或 Naval：长期影响评估
- 或 Human3：能力匹配度验证

### 专家效果评分体系

#### 个人评分（每位专家）

```
专家评分 = 平均满意度×0.4 + 任务完成率×0.3 + 响应速度×0.2 + 用户复用率×0.1

等级：
- 🏆 金牌专家（>4.5分）：优先派遣
- 🥈 银牌专家（3.5-4.5分）：正常派遣
- 🥉 铜牌专家（<3.5分）：减少派遣，检查提示词
```

#### 组合评分（专家搭配）

```
组合评分 = 平均满意度 + 观点互补性 + 冲突解决率

黄金组合标准：
- 使用次数 ≥ 3次
- 平均满意度 ≥ 4.5分
- 无重大冲突记录
```

## 六、迭代优化流程

### 场景：极复杂问题的多轮处理

**用户问题**："AI时代的教育改革应该怎么设计"

**第一轮派遣**：
```
复杂度评估：48分（极复杂）
派遣：Researcher + Thinker

Researcher输出：当前AI教育技术和市场分析
Thinker输出：教育本质的重新思考
```

**调度员分析中间结果**：
> 第一轮分析显示，需要更具体的实施方案和决策模拟。

**第二轮派遣**：
```
补派：Methodology（设计实施框架）
     Decision（模拟政策决策过程）
```

**第三轮综合**：
```
补派：Coach（检视方案盲区）
```

**最终输出**：综合4位专家观点，给出完整方案

## 七、用户反馈收集

### 显式反馈（直接询问）

在专家回答后，询问：
> 这次分析对你有帮助吗？（1-5分）
> 哪位专家的观点最有价值？
> 还希望听到哪些角度的分析？

### 隐式反馈（行为推断）

- **高满意度信号**：追问细节、表示感谢、分享对话
- **低满意度信号**：重复提问、要求换专家、长时间不回复
